# Android 自动化测试用例设计

## 概述

将测试计划中的场景拆解为可执行的测试用例，每个用例的步骤描述紧密对应 Midscene.js API 调用。

## 设计原则

1. **一步一动作**：每个操作步骤对应一个 Midscene 方法调用
2. **自然语言即指令**：步骤描述可直接作为 `aiAct()` 或即时操作的参数
3. **独立可执行**：每个用例可独立运行
4. **断言明确**：每个用例至少一个验证点

## 用例模板

每个用例遵循以下结构：

```markdown
### TC-001：用例标题

| 属性 | 值 |
|------|-----|
| 用例ID | TC-001 |
| 优先级 | P0/P1/P2 |
| 模块 | 所属功能模块 |
| 前置条件 | 需要的环境/数据/状态 |
| 依赖用例 | 无 / TC-xxx |

**操作步骤：**

| 步骤 | Midscene API | 描述 | 预期结果 |
|------|-------------|------|---------|
| 1 | agent.launch('包名') | 启动应用 | 应用成功启动 |
| 2 | agent.aiWaitFor('目标元素出现') | 等待首页加载 | 看到首页内容 |
| 3 | agent.aiTap('目标按钮') | 点击目标按钮 | 跳转到目标页面 |
| 4 | agent.aiAssert('验证点描述') | 验证结果 | 断言通过 |

**后置清理：**
- agent.press('home') — 返回桌面
```

## 用例覆盖矩阵

确保覆盖以下维度：

### 正向场景
- 核心业务流程完整走通
- 所有主要入口可达

### 负向场景
- 非法输入处理
- 网络异常处理（如可模拟）
- 权限拒绝处理

### 边界场景
- 空数据、超长文本
- 快速点击、重复操作
- 横竖屏切换

## Midscene 方法选择指南

| 步骤类型 | 推荐方法 | 示例描述 |
|---------|---------|--------|
| 复杂多步交互 | `agent.aiAct()` | "登录：输入账号密码，点击登录" |
| 单次点击 | `agent.aiTap('按钮')` | "点击确认按钮" |
| 文本输入 | `agent.aiInput('搜索框', '关键词')` | "在搜索框输入'测试'" |
| 页面滚动 | `agent.aiScroll(方向) | "向下滚动到评论区" |
| 等待某状态 | `agent.aiWaitFor('描述')` | "等待商品列表加载完成" |
| 断言验证 | `agent.aiAssert('描述')` | "页面上有'提交成功'提示" |
| 提取数据 | `agent.aiQuery('{...}')` | "提取商品名称和价格列表" |
| 应用控制 | `agent.launch/terminate()` | "启动/关闭淘宝应用" |

## 注意事项

- 步骤描述尽量用中文自然语言，Midscene 的 AI 会理解
- 描述要具体，避免模糊词（❌ "操作一下" ✅ "点击底部'我的'标签"）
- 对于需要验证多个元素的场景，拆分为多个独立用例
- 考虑 `aiActionContext`：提前声明通用弹窗处理策略
